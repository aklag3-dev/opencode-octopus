Real boundaries, not prose pleas.

Features

Orchestration as code.

Smart defaults, strong boundaries, and tool-level enforcement — not prose pleas.

🎯

Contract-First Delegation

Every agent gets success criteria, scope boundaries, output format, and escalation triggers before starting.

⚡

Three Orchestration Modes

Sequential pipelines, parallel fan-outs, review loops. Mix and match for any workflow.

🛡️

Quality Gates

Enforced checks between every stage: Research→Design, Design→Build, Build→Deploy, Deploy→Market.

📁

Filesystem Ground Truth

All outputs written to disk. Status files prevent loading incomplete work. Survives context resets.

🎨

Five Agent Colors

Yellow, Red-Orange, Blue, Green, Purple — each with distinct role, tools, and permissions.

🧠

Venture Metaskill

Discover micro-tool opportunities, build, launch, and evaluate. Hunt mode + Harvest mode.

Orchestration

Three modes. Infinite combinations.

Choose the right pattern for your workflow. Patterns compose for complex pipelines.

📋 Sequential Pipeline

Researcher → Designer → Maker → Marketer

Each stage depends on the previous. Use when work is naturally sequential.

🔀 Parallel Fan-out

Manager → [Agent A, Agent B, Agent C] → Manager

Independent tasks run simultaneously. Manager collects and synthesizes.

🔄 Review Loop

Builder → Reviewer → Builder → ... (max 5 rounds)

Iterative refinement between two agents. Use when quality matters most.

Quick Start

Try it today.

Three ways to use Octopus with OpenCode.

1

🐙 Terminal Command

Install octopus as a global command. Works from any directory.

git clone https://github.com/anomalyco/opencode-octopus
cd opencode-octopus && bash install.sh
source ~/.zshrc
octopus
2

📁 Project Mode

Clone and open with OpenCode. OPENER.md loads automatically.

git clone https://github.com/anomalyco/opencode-octopus
cd opencode-octopus && opencode
3

📄 Any LLM Terminal

Add octopus.md to your system prompt or project context. Works with Claude Code, Codex CLI, Gemini CLI, and more.

# In your opencode.json:
"instructions": ["octopus.md"]
Safety

Zero config collision.

Octopus ships no hooks, no settings changes, no config overrides.

🔒

No Global Installation

Installs under ~/.config/opencode/octopus/. Never touches your existing config.

🛡️

Least Privilege

Each agent has exactly the tools it needs. Tool allowlists are the enforcement layer.

🚫

No Autonomous Distribution

The Marketer drafts; a human sends. This is a permanent architectural gate, not a setting.
